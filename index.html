The git-worklow

When working collaboratively on a project, using git, there are some essential steps to be followed.
First one has to create a lokal 'clone' of the 'master'-folder. The master is normally stored in an online repository. Cloning the master with the command 'git clone' creates a local copy of the folder, holding the project files.
After manipulating the local files in any manner, there are three steps to be taken, if the changes are to be uploaded into the repository. Using the command 'git add', while specifying one or more files, those files will be marked, to be part of the next commitment to the server. To add all files in the working directory, the command 'git add *.*' can be used.
To actually create a new commitment the command 'git commit' is used. With that, all files formerly added will be saved as the latest commitment. That can now be uploaded to the repository with the command 'git push'. The master on the repository will be updated with the changes and the name of the committing colleague will be saved in the change-log.
If one wants to download the latest changes from the repository instead, the command 'git pull' will synchronize the local clone to match the master.
